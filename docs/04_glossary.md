# 04_glossary.md

## 1. Назначение документа

Документ фиксирует каноническую терминологию V1.

## 2. Термины

### Service
Наблюдаемый платёжный сервис или критичная интеграционная точка.

### Current status
Текущее нормализованное прикладное состояние сервиса.

### Authoritative status source
Внешний источник, который считается источником истины для `current_status`.

### Status source key
Внешний ключ сопоставления сервиса с авторитетным источником состояния.

### Metrics source key
Внешний ключ сопоставления сервиса с Prometheus / metrics wiring.

### ServiceOwner role
Фиксированная прикладная роль V1.

### Ownership relation
Связь между конкретным пользователем и конкретным сервисом.

### Owner-scoped access
Набор действий, доступных только пользователю с ролью `ServiceOwner` и ownership relation к конкретному сервису.

### Maintenance
Единственный manual status V1, устанавливаемый административной ролью.

### Unknown
Статус, используемый когда система не может доверенно определить актуальное состояние сервиса.

### Role-safe diagnostics subset
Безопасный поднабор diagnostics, доступный обычным ролям.

### Admin diagnostics
Расширенный diagnostics set, доступный только административным представлениям и admin-role.

### Controlled seed
Контролируемый server-side путь подготовки данных, не заменяющий обычные пользовательские use cases.

### Backend-only administrative flow
Операционный путь выполнения административного действия вне публичного UI.

### Data-quality issue
Нарушение обязательных invariants данных без автоматического сокрытия или silent repair.

### Stale data
Данные, у которых истёк допустимый срок актуальности.
