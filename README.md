# first_app_for_bank

Учебно-практический проект внутренней системы **Manage MegaPayment Service**.

## Что это

Проект описывает и постепенно реализует внутреннее веб-приложение для:
- наблюдения за состоянием платежных сервисов;
- просмотра эксплуатационного контекста по сервисам;
- анализа деградации, недоступности и динамики работы сервисов;
- поддержки ручной эксплуатационной реакции и эскалации.

## Главные документы

- `PROJECT_FULL.md` — единый полный документ проекта для чтения одним файлом;
- `docs/01_project_foundation.md` — основной рабочий консолидированный документ проекта;
- `docs/07_prometheus_metrics_dynamics.md` — дополнительный документ по динамике метрик и роли Prometheus.

## Текущее состояние

На текущем этапе в проекте уже есть:
- описание проекта по 18 блокам;
- базовый UI-каталог сервисов и карточка сервиса;
- API для получения списка сервисов и карточки по `id`;
- первый слой persistence через **EF Core + PostgreSQL**;
- fallback на in-memory режим, если PostgreSQL не включен;
- базовый docker-compose для локального Postgres;
- начальная миграция и seed стартовых данных.

## Локальный запуск с PostgreSQL

### 1. Поднять БД

```bash
docker compose up -d
```

### 2. Включить PostgreSQL для приложения

Для локального запуска удобно передать настройки через переменные окружения:

```bash
Infrastructure__UsePostgres=true
ConnectionStrings__DefaultConnection="Host=localhost;Port=5432;Database=first_app_for_bank;Username=postgres;Password=postgres"
```

### 3. Запустить приложение

```bash
dotnet run --project src/FirstAppForBank.Web
```

При старте приложение применит миграции и выполнит seed стартовых данных.

## Запуск без PostgreSQL

Если `Infrastructure:UsePostgres=false` или строка подключения не задана, приложение автоматически работает через `InMemoryServiceCatalogReader`.

Это удобно для быстрого UI/API запуска и для тестового контура.
